Saya adalah seorang Cyber Security Engineer yang ditugaskan untuk membangun sebuah aplikasi web Cyber Asset Management & Tracking System.

Latar Belakang Masalah:
Selama ini, manajemen aset di divisi Cyber masih dilakukan secara manual tanpa pencatatan yang jelas. Banyak divisi lain yang meminjam aset (seperti hardware, perangkat uji, dongle, laptop, dll.) tanpa serah terima resmi. Akibatnya, saat aset hilang, kami kesulitan melacak keberadaan aset tersebut dan siapa personil yang bertanggung jawab (broken accountability).

Tujuan Aplikasi:
Membangun sistem inventaris berbasis web untuk mencatat seluruh aset divisi Cyber, melacak status dan lokasinya secara real-time, mengotomatisasi proses menggunakan QR Code, merekam koordinat geografis, serta mendokumentasikan setiap aktivitas peminjaman.

Spesifikasi Teknis Database:
Untuk tahap pengembangan awal/internal ini, saya wajib menggunakan SQLite3 sebagai database utamanya.

Mohon bantu saya untuk menyusun cetak biru (blueprint) aplikasi ini berdasarkan rincian kebutuhan fitur dan struktur menu berikut:

1. Spesifikasi Fitur Lengkap (Feature Requirements)
Modul Manajemen & Integrasi QR Code: Generasi otomatis QR Code unik, fitur cetak label, dan scan menggunakan kamera/webcam untuk transaksi cepat.

Modul Pelacakan Lokasi (Geolocation Tracking): Menangkap koordinat GPS/IP Address via HTML5 Geolocation API saat Check-in/Check-out, serta mencatat riwayat lokasi terakhir aset.

Modul Inventaris Aset: Pencatatan detail (Nama, Kategori, Serial Number, Status, Lokasi Fisik).

Modul Peminjaman & Pengembalian: Form peminjaman digital dan alur persetujuan (approval workflow) oleh Admin Cyber.

Modul Riwayat & Audit Log: Rekam jejak peminjaman aset dari waktu ke waktu dan Immutable Audit Log untuk kebutuhan investigasi keamanan.

Modul Notifikasi: Sistem pengingat otomatis untuk aset yang terlambat dikembalikan (overdue).

2. Struktur Menu Sidebar (Navigation & UI Structure)
Saya ingin struktur menu sidebar yang rapi dan responsif, dengan pembagian hak akses (Role-Based Access Control) sebagai berikut:

- Dashboard (Menu Utama): Menampilkan ringkasan total aset, jumlah aset yang sedang dipinjam, grafik status aset, dan alert aset overdue. (Akses: Admin & User)

- Scan QR Code (Quick Action): Akses cepat ke kamera untuk langsung melakukan scanning aset saat proses peminjaman/pengembalian di tempat. (Akses: Admin & User)

- Manajemen Aset (Asset Inventory): Daftar seluruh aset, tambah aset baru, generate QR code, dan edit data aset. (Akses: Khusus Admin)

-Transaksi Peminjaman (Loan Tracking):

Form/Request Pinjam: Untuk user luar mengajukan peminjaman. (Akses: User)

Persetujuan (Approval): Daftar antrean pengajuan yang butuh konfirmasi. (Akses: Khusus Admin)

Daftar Peminjaman Aktif: Memantau barang apa saja yang sedang di luar beserta nama peminjamnya. (Akses: Admin & User)

Peta Geolocation (Asset Map Tracking): Visualisasi peta (map view) yang menampilkan titik koordinat terakhir aset-aset yang sedang dipinjam berdasarkan data GPS tracking. (Akses: Khusus Admin)

Manajemen Pengguna (User Management): Mengelola data pengguna (nama, divisi, kontak) dan pembagian role (Admin Cyber vs Peminjam/User Umum). (Akses: Khusus Admin)

Log Audit & Laporan (Audit Logs): Halaman riwayat seluruh aktivitas sistem (siapa melakukan apa dan kapan) yang tidak bisa dimodifikasi. (Akses: Khusus Admin)

3. Output yang Saya Butuhkan dari Anda (AI):
Rancangan Skema Database (SQLite3): Struktur tabel yang ideal (Tabel assets, users, loans, audit_logs, locations) yang mendukung semua fitur dan menu di atas. Berikan contoh query CREATE TABLE-nya.

Alur Kerja Pengguna (User Workflow): Alur integrasi menu sidebar, scanning QR Code, hingga perekaman koordinat geolocation dari awal sampai akhir.

Aspek Keamanan (Cyber Security Best Practices): Penerapan Role-Based Access Control (RBAC) pada menu sidebar agar user tidak bisa menembak URL menu Admin secara ilegal, serta pengamanan database SQLite3.

Rekomendasi Tech Stack & Library: Saran teknologi (Frontend & Backend) yang cocok dengan SQLite3, beserta library/package web untuk QR Code, kamera, dan visualisasi peta (maps).

Tolong berikan penjelasan yang terstruktur, rapi, dan mudah diimplementasikan langkah demi langkah.
